• SpatialOS is a managed cloud service to host any synchronous multiplayer games. It is not licenced software.
  • It continuously improves, and users automatically benefit
  • It can be used for games on PC/Xbox/PS4/Mobile.
  • Compared with games that exist today, it enables new gameplay, and fewer gameplay trade-offs.
    • Games can run across many game engine instances and/or servers, enabling new gameplay resulting from combinations of larger seamless worlds, more AI with greater complexity, millions of interactable objects & rigid bodies, and more complex gameplay systems
    • It provides networking and area of interest management, enabling 10,000’s of players to connect to a single game world, and a higher density of players and object interactions
    • SpatialOS allows for near-instant messages within one large seamless world, enabling players to see and interact with each other over larger distances
    • It is trivial to add persistence elements, meaning when something happens in the game world the results of that change permanently affect the world, even when players leave the game.
    • It enables interaction between players and systems in different, distinct game worlds
    • It enables different platforms to play and interact in the same game world. For example, players on mobile controlling a drone to help their teammates playing on PC
  • It enables fast creation and testing of new games
    • You can deploy and test at scale quickly
    • We provide an easy way to distribute your game to players for playtests
    • We provide a service that lets you test your game’s performance using fake players
  • It has hosting locations in all major gaming countries around the world (including China)
  • It can easily and quickly scale the amount of compute resources to match your games needs
  • You cannot use your own private servers to host the game. You can use them for ancillary services.
  • It does not provide most ancillary services required for a game. However, it is easy to integrate your own or a 3rd party services, with our SDKs.
  • It is not a game engine and does not provide game logic. When using SpatialOS you can use a game engine to write your game logic. You can integrate any game engine with SpatialOS using our SDKs, and we provide our own integration for Unity and Unreal.
  • It works for development teams of all sizes and in any region. They will need to learn new APIs and some basic knowledge of distributed computing, but don’t need expert knowledge in distributed computing.
  • You don’t always need to start a-fresh when making a SpatialOS game. In some cases you can quickly and easily port an existing game.

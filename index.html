• SpatialOS is a managed cloud service to host any synchronous multiplayer games. It is not licensed software.

    • By managed cloud service, we mean SpatialOS continuously improves, and users automatically benefit without any additional costs

  • SpatialOS is suitable for any online synchronous multiplayer game
  • This includes both persistent, large-scale multiplayers AND smaller, match-based arena games
  • It can be used for games on PC, Xbox, PS4 and Mobile (iOS and Android)
  • You can port in-development or live games to SpatialOS. This would enable you to grow an existing match-based shooter into a larger, 1000-player shooter, or even into a persistent MMO
  • Games can run across many game engine instances and/or servers. This enables game worlds that are much larger in size, have more players (1,000s in a single game world), have a greater number of complex AI and gameplay systems (e.g. weather, ecologies), have many physical objects and run at a higher fidelity (enabling new gameplay mechanics, such as hi-fidelity combat)
  • You must use SpatialOS’ networking stack, and cannot use your own

    • It provides networking and area of interest management (only send relevant state changes to each player), enabling 10,000’s of players to connect to a single game world, and a higher density of players and object interactions

    • SpatialOS allows for near-instant messages within one large seamless instance, enabling players to see and interact with each other over larger distances

    • It is trivial to add persistence elements, meaning when something happens in the game world the results of that change permanently affect the world, even when players leave the game. This data is stored in SpatialOS' scalable, in-memory database, so you do not need to add your own back-end

      • You can mix both persistent and temporary (session-based) elements in a single game world

    • It enables interaction between players and systems in different, distinct game worlds. For example, players in one world could banish a daemon, causing it to reappear in another world

    • It enables different platforms to play and interact in the same game world. For example, players on mobile controlling a drone to help their teammates playing on PC

    • It enables fast creation and testing of new games

      • You can deploy and test at scale quickly

      • We provide an easy way to distribute your game to players for playtests (although it does not provide a commercial distribution service)

      • We provide a service that lets you test your game’s performance using fake players

      • We provide basic logging & metrics to understand the health and performance of your deployment for devops purposes

    • It has hosting locations in all major gaming countries around the world (including China)

    • It can easily and quickly scale the amount of compute resources to match your games needs

    • You cannot use your own private servers to host the game. You can use them for ancillary services.

    • It does not provide most ancillary services required for a game. However, it is easy to integrate your own or a 3rd party services, such as player-analytics and leaderboards, with our SDKs.

    • It is not a game engine and does not provide game logic (e.g. for physics, or other gameplay systems). When using SpatialOS you can use a game engine to write your game logic, although SpatialOS will provide the compute to facilitate more complex systems than otherwise possible

    • You can integrate any game engine with SpatialOS using our SDKs, and we provide our own integration for Unity and Unreal.
